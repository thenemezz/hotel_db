Информационная система гостиничного комплекса

Информационная система гостиничного комплекса представляет собой веб-приложение, разработанное на стеке технологий Python + Flask + PostgreSQL, для автоматизации управления гостиничным бизнесом.

Основные возможности системы:

  • Управление отелями и номерным фондом
  • Бронирование номеров для гостей и организаций
  • Учет дополнительных услуг и их оплаты
  • Ведение базы данных гостей и организаций
  • Учет договоров с организациями и применение скидок
  • Обработка и анализ жалоб клиентов
  • Статистика и аналитика по работе гостиничного комплекса

Технические характеристики:

  • Backend: Python, Flask
  • База данных: PostgreSQL
  • ORM: Прямые запросы через psycopg2
  • Frontend: HTML, CSS, JavaScript, Bootstrap 5
Описание гостиничного комплекса

Гостиничный комплекс состоит из нескольких зданий-гостиниц (корпусов). Каждый корпус имеет ряд характеристик:

  • Класс отеля (от двух до пяти звезд)
  • Количество этажей в здании
  • Общее количество комнат
  • Местность номеров (одно-, двух-, трехместные и т.д.)
  • Наличие служб быта:
    • Ежедневная уборка номера
    • Прачечная
    • Химчистка
  • Питание:
    • Рестораны
    • Бары
  • Развлечения:
    • Бассейн
    • Сауна
    • Бильярд

От типа корпуса и местности номера зависит сумма оплаты за него. Дополнительные услуги (химчистка, стирка, питание, развлечения) предоставляются за отдельную плату.

Работа с организациями

С крупными организациями заключаются договора, позволяющие бронировать номера с большими скидками:

  • Туристические фирмы
  • Организации, занимающиеся проведением:
    • Международных симпозиумов
    • Конгрессов
    • Семинаров
    • Карнавалов и т.д.

Система позволяет бронировать номера на определенное время вперед для групп людей. Предпочтительно группы людей от одной организации не расселять по разным этажам.

В брони указывается класс отеля, этаж, количество комнат и общее количество людей. Броня может быть отменена за неделю до заселения.

Виды запросов в информационной системе
  1. Получить перечень и общее число фирм, забронировавших места в объеме, не менее указанного, за весь период сотрудничества, либо за некоторый период.
  2. Получить перечень и общее число постояльцев, заселявшихся в номера с указанными характеристиками за некоторый период.
  3. Получить количество свободных номеров на данный момент.
  4. Получить сведения о количестве свободных номеров с указанными характеристиками.
  5. Получить сведения о конкретном свободном номере: в течение какого времени он будет пустовать и о его характеристиках.
  6. Получить список занятых сейчас номеров, которые освобождаются к указанному сроку.
  7. Получить данные об объеме бронирования номеров данной фирмой за указанный период, и каким номерам отдавались предпочтения.
  8. Получить список недовольных клиентов и их жалобы.
